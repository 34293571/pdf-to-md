## H2: Limits & Supported PDFs {#limits}

PDF a MD works best with **text-based PDFs** — files where you can select and copy text in a PDF reader.

**Supported**

- PDFs with selectable text
- Documents up to **10MB** and **50 pages**
- Standard, unencrypted PDF files

**Not supported (MVP)**

- Scanned documents and image-only PDFs
- Password-protected or encrypted PDFs
- OCR (may be considered later)

If upload fails, check the error message — it usually means the file exceeds limits or is a scanned PDF.

## H2: FAQ {#faq}

**Q: What does "pdf a md" mean?**
A: It means converting a PDF file to Markdown (`.md`). "PDF a MD" is how many users search for this — similar to "pdf to md" or "pdf to markdown" in English.

**Q: Is PDF a MD free?**
A: Yes. pdfamd.com is free to use with no signup required.

**Q: How is this different from "pdf to md"?**
A: Same thing. PDF a MD is our brand name; pdf to md describes the conversion. Both refer to turning PDF content into Markdown format.

**Q: Does it work with scanned PDFs?**
A: No. This version only supports PDFs that already contain extractable text. Scanned pages need OCR, which we don't offer yet.

**Q: What's the maximum file size and page count?**
A: 10MB per file and up to 50 pages. These limits keep conversions fast and reliable.

**Q: Is my PDF stored on your server?**
A: No. Files are processed in memory for conversion and are not saved to disk or cloud storage afterward.

**Q: Why is the first conversion sometimes slow?**
A: Our conversion service may need a moment to wake up after idle time. Later requests in the same session are usually faster.

**Q: What Markdown flavor do you output?**
A: We aim for common Markdown compatible with most editors — headings, paragraphs, and basic structure from your PDF text.

**Q: Can I use the output in Obsidian / Notion / VS Code?**
A: Yes. Download the `.md` file and open it in any Markdown-compatible app.
